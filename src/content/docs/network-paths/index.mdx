# Network Paths

Network path choice is usually the most decisive architecture step in remote telemetry. Once the backhaul model is chosen, the rest of the stack tends to follow. This section keeps that decision tied to field conditions instead of product preference, and it works best after the asset type and operating model are already clear.

## Network review flow

<Steps>
	1. Define site geography, asset density, and maintenance access.
	2. Confirm latency tolerance, buffering needs, and expected data volume.
	3. Compare coverage confidence, power burden, and deployment complexity.
	4. Choose the path that best fits the field operating model, not the one with the best slide deck.
</Steps>
